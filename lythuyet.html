Phần 1: Lời giải Lý thuyết (40 điểm)

Bản dưới đây trình bày chi tiết theo đúng 4 câu hỏi trong đề, có ví dụ mã minh họa và giải thích cách áp dụng thực tế.

Câu 1: Cách sử dụng Media Queries để làm website responsive và tích hợp JavaScript

Khái niệm: Media Query là kỹ thuật trong CSS giúp áp dụng bộ style theo điều kiện thiết bị, phổ biến nhất là chiều rộng màn hình. Mục tiêu là một bộ HTML nhưng hiển thị phù hợp cho desktop, tablet và mobile.

Nguyên tắc thiết kế responsive hiệu quả:

  • Ưu tiên bố cục linh hoạt bằng Grid/Flex thay vì kích thước cố định.
  • Dùng đơn vị tương đối như %, rem, vw để giao diện co giãn tốt.
  • Chia breakpoints theo nội dung thực tế, không theo máy cố định.

Ví dụ CSS:

/* Desktop mặc định */
.container {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: 16px;
}

/* Tablet */
@media (max-width: 992px) {
  .container {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* Mobile */
@media (max-width: 600px) {
  .container {
    grid-template-columns: 1fr;
  }
  .sidebar {
    display: none;
  }
}

Tích hợp JavaScript khi thay đổi kích thước: CSS xử lý giao diện, còn JavaScript xử lý hành vi. Có thể dùng window.matchMedia() để đổi logic theo màn hình, ví dụ tắt autoplay carousel trên mobile để tiết kiệm pin và dữ liệu.

const mobileQuery = window.matchMedia('(max-width: 600px)');

function updateBehavior() {
  if (mobileQuery.matches) {
    console.log('Đang ở mobile: giảm animation nặng');
  } else {
    console.log('Đang ở màn hình lớn: bật đầy đủ hiệu ứng');
  }
}

mobileQuery.addEventListener('change', updateBehavior);
updateBehavior();

Kết luận: Responsive tốt nhất khi kết hợp "CSS cho trình bày" và "JS cho hành vi", giúp website vừa đẹp vừa mượt trên nhiều thiết bị.

Câu 2: Quy trình deploy website lên hosting bằng FTP và công cụ hỗ trợ như VS Code extension

Quy trình triển khai cơ bản:

  1. Chuẩn bị domain, hosting và mã nguồn sạch (kiểm tra đường dẫn file tương đối).
  2. Trỏ DNS domain về hosting (qua nameserver hoặc A record).
  3. Tạo tài khoản FTP trong cPanel: host, username, password, port.
  4. Cài extension FTP trên VS Code (ví dụ SFTP/FTP-Simple), cấu hình kết nối.
  5. Upload mã nguồn lên thư mục chạy web như public_html.
  6. Kiểm tra website online, sửa lỗi rồi đồng bộ lại file.

Vai trò của extension FTP trong VS Code: giúp upload nhanh file đang sửa, tiết kiệm thời gian so với thao tác thủ công từng lần qua trình FTP riêng.

Phân biệt Shared Hosting và VPS:

  • Shared Hosting: chi phí thấp, dễ dùng, phù hợp website nhỏ; nhược điểm là tài nguyên dùng chung nên có thể chậm khi server đông người.
  • VPS: có tài nguyên riêng, chủ động cấu hình server, hiệu năng cao hơn; nhược điểm là cần kỹ năng quản trị hệ thống và chi phí cao hơn.

Rủi ro nếu domain không renew kịp:

  • Website ngừng truy cập ngay khi domain hết hạn.
  • Email theo domain ngừng hoạt động.
  • Nguy cơ mất tên miền vào tay người khác nếu bị mua lại.
  • Ảnh hưởng uy tín thương hiệu và trải nghiệm người dùng.

Khuyến nghị: bật tự động gia hạn (auto-renew), gia hạn sớm 1-3 năm với domain quan trọng và theo dõi email cảnh báo từ nhà đăng ký.

Câu 3: Promise và async/await, ví dụ fetch API và cách debug lỗi

Promise là gì? Promise đại diện cho kết quả của tác vụ bất đồng bộ, có 3 trạng thái: pending, fulfilled, rejected.

async/await là gì? Là cú pháp giúp viết mã bất đồng bộ theo kiểu tuần tự, dễ đọc hơn chuỗi .then().

Ví dụ fetch dữ liệu từ API:

async function tảiDanhSáchNgườiDùng() {
  try {
    const response = await fetch('https://jsonplaceholder.typicode.com/users');

    if (!response.ok) {
      throw new Error('Lỗi HTTP: ' + response.status);
    }

    const dữLiệu = await response.json();
    console.log('Dữ liệu nhận được:', dữLiệu);
  } catch (lỗi) {
    console.error('Không thể tải dữ liệu:', lỗi.message);
  }
}

tảiDanhSáchNgườiDùng();

Cách debug hiệu quả:

  • Dùng console.log() ở từng bước để biết dữ liệu đang đi tới đâu.
  • Mở tab Network trong DevTools để xem URL, status code, payload, thời gian phản hồi.
  • Test API bằng Postman trước khi ghép vào giao diện để tách lỗi frontend/backend.
  • Kiểm tra lỗi phổ biến: sai URL, thiếu token, CORS, parse JSON sai định dạng.

Kết luận: Promise là nền tảng, async/await giúp code dễ bảo trì hơn; kết hợp debug đúng quy trình sẽ giảm thời gian sửa lỗi đáng kể.

Câu 4: Khái niệm Closure trong JavaScript và ứng dụng

Định nghĩa: Closure là cơ chế cho phép hàm con "ghi nhớ" biến trong phạm vi hàm cha, kể cả khi hàm cha đã thực thi xong. Nhờ đó có thể tạo dữ liệu private mà bên ngoài không truy cập trực tiếp được.

Ví dụ counter private:

function tạoBộĐếm() {
  let count = 0; // biến private

  return {
    tăng() {
      count++;
      return count;
    },
    giảm() {
      count--;
      return count;
    },
    lấyGiáTrị() {
      return count;
    }
  };
}

const bộĐếm = tạoBộĐếm();
console.log(bộĐếm.tăng());    // 1
console.log(bộĐếm.tăng());    // 2
console.log(bộĐếm.lấyGiáTrị()); // 2

Ứng dụng với form HTML để xử lý state: Closure giúp gom state trong một module thay vì biến global, ví dụ lưu trạng thái form nhiều bước (step hiện tại, dữ liệu tạm nhập) và chỉ cho phép cập nhật qua hàm điều khiển.

function quảnLýForm() {
  let state = { step: 1, dữLiệu: {} };

  return {
    cậpNhật(field, value) {
      state.dữLiệu[field] = value;
    },
    quaBướcSau() {
      state.step++;
    },
    lấyState() {
      return state;
    }
  };
}

Lợi ích: bảo mật dữ liệu nội bộ, giảm xung đột biến toàn cục, tăng khả năng tái sử dụng và kiểm soát luồng xử lý.
